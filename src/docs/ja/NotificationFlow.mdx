# 告知フロー

`NotificationManager` は当選告知のタイミング判定を行います。**ロジックとイベント発火のみ**を担当し、演出（UI/アニメーション/サウンド）は開発者に委ねます。

## 告知タイプ

| タイプ | タイミング | 説明 |
|--------|----------|------|
| `PRE_SPIN` | スピン開始前 | 先告知 |
| `POST_SPIN` | リール停止後 | 後告知 |
| `NEXT_BET` | 次ゲームBET時 | 次ゲームBET告知 |
| `LEVER_ON` | レバーON時 | 自ゲームレバーON告知 |

## フロー図

```mermaid
sequenceDiagram
    participant GCM as GameCycleManager
    participant IL as InternalLottery
    participant NM as NotificationManager
    participant Dev as 開発者コールバック

    GCM->>IL: draw(gameMode)
    IL-->>GCM: WinningRole
    GCM->>NM: check(LEVER_ON, winningRole)
    NM-->>Dev: onNotification(payload)
    GCM->>NM: check(PRE_SPIN, winningRole)
    NM-->>Dev: onNotification(payload)
    Note over GCM: リール回転・停止
    GCM->>NM: check(POST_SPIN, winningRole, spinResult)
    NM-->>Dev: onNotification(payload)
    Note over GCM: 次ゲーム
    GCM->>NM: check(NEXT_BET)
    NM-->>Dev: onNotification(payload)
    NM->>NM: clearPreviousWin()
```

## 使用例

```tsx
import { useNotification } from 'reeljs';

const { status, lastPayload, acknowledgeNotification } = useNotification({
  enabledTypes: { PRE_SPIN: true, POST_SPIN: true },
  targetRoleTypes: ['BONUS'],
});
```
